🐛 発見された問題

問題1: URLパラメータが消える

項目 期待値 実際の値 状態
元のURL data=!3e3!8j1755252000 含まれている OK
リダイレクト後 !3e3 (公共交通機関) 消えている NG
リダイレクト後 !8j (時刻指定) 消えている NG

原因: Google Mapsがリダイレクト時にURLパラメータを再構築し、一部のパラメータが失われる

問題2: 詳細ステップセレクタが機能しない

試したセレクタ: 1. //div[@class="cYhGGe"] → 見つからず 2. //div[contains(@class, "directions-mode-step")] → 見つからず 3. //div[contains(@class, "section-directions-trip")] → 見つからず

原因: Google MapsのHTML構造が変更され、古いセレクタが使えない

問題3: extract_route_steps()の無限待機

詳細ステップが見つからないため、WebDriverWaitが20秒待機を繰り返し、最終的にタイムアウト

✅ 解決策

解決策1: URLパラメータの確認方法を改善

  • リダイレクト後のURLではなく、ページの状態で判定
  • ルート要素の存在 = パラメータ適用済みと判断
  • すでに実装済み（check_needs_setup）だが、判定ロジックを調整必要

解決策2: セレクタを最新のものに更新

// 新しいセレクタ候補 //div[@data-trip-index] // ルート全体 //div[contains(@class, "trip-details")] // 詳細部分 //span[contains(@class, "transit-mode")] // 交通手段

解決策3: タイムアウト処理の改善

  • WebDriverWaitのタイムアウトを短く（5-10秒）
  • 要素が見つからない場合は即座にフォールバック
  • サマリー情報だけでも返す

📝 結論

タイムアウトの直接原因:

  1. URLパラメータが正しく認識されず、手動設定モードに入る
  2. 詳細ステップ抽出で古いセレクタを使用
  3. 要素が見つからず20秒の待機を繰り返す
  4. 合計で2分を超えてタイムアウト

推奨対応:

  • 🔧 セレクタを現在のGoogle Maps構造に合わせて更新
  • ⏱️ タイムアウトを短くして、失敗時は即座にサマリー情報を返す
  • 📊 詳細が取れなくても、基本情報（時間・料金）は確実に返す
